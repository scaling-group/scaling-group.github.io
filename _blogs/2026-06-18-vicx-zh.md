机器人操作常常被描述成一个控制问题，但在真实场景中，它同样是一个泛化问题。机器人需要从视觉上下文中理解任务，判断接下来应该发生什么，再把这个想象出的未来转化为自身能够执行的动作。每一部分单独来看都不容易；当它们组合在一起时，系统往往会变得脆弱。

在最近的工作 [**VICX: Generalizable Robot Manipulation via Video Generation and In-Context Operator Network**](https://arxiv.org/abs/2606.12028) 中，我们从 in-context operator learning 的角度研究这个问题。项目主页在这里：[https://scaling-group.github.io/vicx/](https://scaling-group.github.io/vicx/)。

VICX 的核心想法，是把高层视觉规划和低层执行落地拆开。一个冻结的视频生成模型负责预测机器人接下来在视觉上应该如何运动；一个上下文算子网络则把生成的视频计划翻译成可执行的机器人状态轨迹。这样，我们可以把视频生成模型的语义灵活性，与机器人控制所需要的 embodiment-specific 精度结合起来。

## 为什么使用视频生成？

视频模型天然适合表达“场景接下来应该发生什么”。给定当前观测和任务提示，它可以生成一段短时未来视频，展示一个合理的任务推进过程。对机器人操作来说，这很有吸引力，因为许多任务用视觉轨迹表达比用低层状态序列表达更自然。

例如，“打开抽屉”并不只是到达某个目标坐标。它包括接近把手、建立接触、沿正确方向拉动，以及在抽屉移动到位后停止。视频模型可以把这种时间结构编码成一个视觉计划。

但是，视频本身还不是可执行策略。它可以展示一个看起来合理的运动过程，但机器人仍然需要状态或动作序列。视觉规划和具体 embodiment 执行之间的 gap，正是 VICX 要解决的问题。

## 从视频到轨迹

VICX 的第二个组件是 **V2T-ICON**，即 Video-to-Trajectory In-Context Operator Network。它的作用是把生成的视频翻译成机器人状态轨迹。

这里的关键设计，是 V2T-ICON 通过上下文进行这种翻译。它不需要针对每个新任务或新 embodiment 重新微调，而是接收检索得到的 image-state reference 作为 prompt examples。这些参考样例告诉模型，在当前设置下，视觉中的机械臂形态如何对应到机器人状态。

从这个角度看，V2T-ICON 把视觉到状态的 grounding 看成一个算子推断问题：

- 生成的视频给出 query visual trajectory。
- 检索到的 image-state pairs 提供 in-context calibration。
- 模型在不更新参数的情况下预测可执行状态轨迹。

这与 **In-Context Operator Networks (ICON)** 的整体思想一致：适应新问题应尽可能通过上下文完成，而不是通过重新训练完成。

## 闭环重规划

机器人操作也非常依赖反馈。开环执行很脆弱：如果生成计划不完美，或者执行偏离了预期状态，系统可能会继续沿着失败轨迹走下去。

VICX 使用闭环方式运行。每执行一个预测的轨迹片段后，系统都会观察新的场景，并再次让视频模型进行规划。这样，失败或不完整的执行结果就会变成新的视觉上下文。视频规划器可以根据真实发生的情况调整下一步计划。

这种闭环结构对长时程操作尤其重要，因为小误差会不断累积。VICX 并不要求一次生成的视频完美解决整个任务，而是在视觉规划、上下文 grounding、执行和观察之间反复交替。

## 我们观察到什么？

在 Meta-World 实验中，VICX 的 V2T-ICON 只在三个源任务上训练，并在更广泛的操作任务上评估。在论文报告的闭环 benchmark 中，VICX 达到了 72.2% 的平均成功率，并优于文中的对比方法。

我们还研究了两类泛化：

- **跨任务泛化：** 系统从源任务迁移到未见过的任务语义。
- **跨 embodiment 泛化：** 当机器人外观和场景视觉风格发生变化时，同一 grounding 机制仍然有效。

上下文参考样例在这两个设置中都很重要。借助检索到的 references，模型可以轻量地重新校准视觉机械臂运动与机器人状态之间的映射，而不需要改变模型权重。

## 为什么这对 ICON 重要？

我们关于 ICON 的许多工作从科学计算出发：学习函数、场、图结构或物理状态之间的算子。VICX 提出的问题是，同样的思想是否也能帮助 embodied AI。

在这里，算子不再是 PDE 的解映射，而是 visual-to-state grounding map。输入是生成的视觉轨迹和一小组 image-state examples；输出是可执行轨迹。底层问题仍然熟悉：模型能否从上下文中推断出当前相关的算子，并立刻应用到新的 query 上？

因此，VICX 在课题组更大的研究方向中扮演了一个连接角色。它把 in-context operator learning 与机器人操作、视频生成和闭环 agentic systems 联系起来。更广义地说，它提示我们：in-context operator networks 不仅可以作为科学问题求解器，也可能成为 foundation models 与物理执行之间的接口。

## 展望

VICX 仍然只是一个早期步骤。视频生成模型正在快速进步，但生成视频中仍可能存在物理不一致。随着环境、embodiment 和动作空间变得更复杂，执行 grounding 也会更困难。

与此同时，这种模块化结构是有前景的。更强的视频模型可以改进视觉规划；更好的检索可以改进上下文 grounding；更多样的 embodiment 数据可以让 visual-to-state operator 更鲁棒。这些改进都不需要把整个系统压缩成一个不透明的端到端策略。

我们希望 VICX 能够成为一个有用的桥梁：连接生成式视觉规划与可执行机器人行为，也连接 in-context operator learning 与 embodied AI。
