This page is a demo post for the new `blogs` section. Future blog entries can follow the same front matter:

```yaml
---
title: "Your Blog Title"
layout: blog_post
author: "Author Name"
date: YYYY-MM-DD
abstract: "One short summary paragraph for the listing page."
---
```

## Suggested Structure

Start with a short motivation paragraph that explains the problem or question the post addresses.

Then break the post into a few clear sections:

1. Background or setup
2. Main idea or method
3. Takeaways, caveats, or open questions

## Example Content

Suppose you want to explain a new operator-learning experiment. The blog can present the task setting, describe what changes compared with prior work, and summarize the most important result in plain language before showing equations or figures.

You can also embed images, equations, or code snippets here as needed. The detail page will render normal Markdown content, so the post can be as short or as long as you want.
