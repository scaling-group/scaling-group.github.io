Robot manipulation is often described as a problem of control, but in practice it is also a problem of generalization. A robot must understand a task from visual context, decide what should happen next, and then convert that imagined future into actions that its own body can execute. Each part is difficult on its own; putting them together is where many systems become brittle.

In our recent work, [**VICX: Generalizable Robot Manipulation via Video Generation and In-Context Operator Network**](https://arxiv.org/abs/2606.12028), we study this problem through the lens of in-context operator learning. The project page is available here: [https://scaling-group.github.io/vicx/](https://scaling-group.github.io/vicx/).

The central idea is to separate high-level visual planning from low-level execution grounding. A frozen video generation model predicts what the robot should visually do next, while an in-context operator network translates the generated visual plan into executable robot-state trajectories. This gives us a modular way to combine the semantic flexibility of video generation with the embodiment-specific precision needed for robot control.

## Why Video Generation?

A video model is naturally good at representing "what should happen" in a scene. Given the current observation and a task prompt, it can generate a short future video showing a plausible task progression. For manipulation, this is attractive because many tasks are easier to express visually than through low-level state sequences.

For example, "open the drawer" is not merely a target coordinate. It involves approaching the handle, making contact, pulling along the correct direction, and stopping once the drawer has moved. A video model can encode this temporal structure as a visual plan.

However, a video alone is not an executable policy. It may show a plausible motion, but the robot still needs a sequence of states or actions. The gap between visual planning and embodiment-specific execution is the key issue VICX is designed to address.

## From Video to Trajectory

The second component of VICX is **V2T-ICON**, a Video-to-Trajectory In-Context Operator Network. Its role is to translate a generated video into robot-state trajectories.

The important design choice is that V2T-ICON performs this translation in context. Instead of fine-tuning the model for every new task or embodiment, it receives retrieved image-state references as prompt examples. These references tell the model how visual arm configurations correspond to robot states in the current setting.

In this sense, V2T-ICON treats visual-to-state grounding as an operator inference problem:

- The generated video provides the query visual trajectory.
- Retrieved image-state pairs provide the in-context calibration.
- The model predicts the executable state trajectory without parameter updates.

This is closely aligned with the broader philosophy of **In-Context Operator Networks (ICON)**: adaptation should happen through context rather than retraining.

## Closed-Loop Replanning

Manipulation also benefits from feedback. Open-loop execution is fragile: if the generated plan is imperfect, or if execution deviates from the intended state, the system may simply continue along a failing trajectory.

VICX instead runs in a closed loop. After executing a predicted trajectory segment, the system observes the new scene and asks the video model to plan again. This turns failure or partial progress into new visual context. The video planner can then adjust the next plan based on what actually happened.

This closed-loop structure is especially important for long-horizon manipulation, where small errors accumulate. Rather than expecting one generated video to solve the entire task perfectly, VICX repeatedly alternates between visual planning, in-context grounding, execution, and observation.

## What We Observed

In Meta-World experiments, VICX is trained with V2T-ICON on only three source tasks and evaluated across a broader set of manipulation tasks. The system reaches a 72.2% average success rate in the reported closed-loop benchmark, outperforming the compared baselines in the paper.

We also studied two forms of generalization:

- **Cross-task generalization:** the system transfers from the source training tasks to unseen task semantics.
- **Cross-embodiment generalization:** the same grounding mechanism remains useful when the robot appearance and scene visual style shift.

The in-context references matter in both settings. With retrieved references, the model has a lightweight way to recalibrate the mapping between visual arm motion and robot state, without changing its weights.

## Why This Matters for ICON

Most of our ICON work begins from scientific computing: learning operators between functions, fields, graphs, or physical states. VICX asks whether the same way of thinking can help with embodied AI.

Here, the operator is not a PDE solution map. It is a visual-to-state grounding map. The input is a generated visual trajectory and a small context set of image-state examples; the output is an executable trajectory. The underlying question is familiar: can a model infer the relevant operator from context and apply it immediately to a new query?

This gives VICX a role in the group's larger research direction. It connects in-context operator learning with robot manipulation, video generation, and closed-loop agentic systems. More broadly, it suggests that in-context operator networks may serve not only as scientific solvers, but also as interfaces between foundation models and physical execution.

## Looking Forward

VICX is still an early step. Video generation models are improving quickly, but generated videos can still contain physical inconsistencies. Execution grounding also becomes more challenging as environments, embodiments, and action spaces become richer.

At the same time, the modular structure is promising. A stronger video model can improve visual planning. Better retrieval can improve in-context grounding. More diverse embodiment data can make the visual-to-state operator more robust. These improvements can be made without collapsing the system into one opaque end-to-end policy.

Our hope is that VICX provides a useful bridge: from generative visual planning to executable robot behavior, and from in-context operator learning to embodied AI.
